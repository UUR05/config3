# Ассемблер для Учебной Виртуальной Машины (УВМ, Вариант №22)

## Описание
Это реализация ассемблера для УВМ по спецификации варианта 22. Ассемблер переводит текстовые команды в промежуточное представление (IR) и кодирует их в байты для тестов. Поддерживает CLI с аргументами для ввода, вывода и теста.

## Требования
- Python 3
- Библиотеки: argparse, re, sys (стандартные)

## Язык ассемблера
Язык основан на спецификации УВМ: мнемоники для операций, аргументы для операндов (B). Синтаксис:
- Команды в uppercase или lowercase (парсер приводит к upper).
- Аргументы через пробел или запятую (например, "LOAD_CONST 629" или "LOAD_CONST, 629").
- Числа: decimal (629) или hex (0x275).
- Комментарии: начинаются с ';'.
- Нет меток/адресов на этапе 1 (добавим позже, если нужно).

### Поддерживаемые мнемоники
- **LOAD_CONST value**: Загрузка константы (A=7, B=value). Размер: 5 байт. Результат: Новый элемент на стеке.
- **READ_MEM offset**: Чтение из памяти (A=61, B=offset). Размер: 2 байт. Результат: Значение по сумме адреса (со стека) + offset на стек.
- **WRITE_MEM**: Запись в память (A=27, без B). Размер: 1 байт. Результат: Значение со стека записывается по адресу со стека.
- **BITREVERSE value**: Унарная операция bitreverse (A=25, B=value?). Размер: 2 байт. Результат: Bitreverse элемента со стека, результат на стек (по спецификации — запись в память? Уточнить).
### Использование
CLI-аргументы:
- python assembler.py input.asm — Ассемблирует в program.bin, выводит количество команд.
- python assembler.py input.asm --test — + Выводит hex-байты как в тестах.
- python assembler.py input.asm -o output.bin — Указывает путь к бинарнику.
